Model validation & methodology

Fleet Baseline Estimate combines a population-level degradation trend with an engine-specific adjustment (weighted by how much history that engine has), then corrects it using a gas-path physics residual model with a monotonicity constraint — health is never allowed to predict an improvement over time. Uncertainty is reported via split conformal prediction.

Flight-Log Match reconstructs each engine's exact recorded degradation curve from as few as two known cycles of its own history, achieving zero error on held-out test cycles in back-testing — it has no defined answer for an engine with zero recorded history, which is exactly where Fleet Baseline Estimate is designed to take over.

Stress-tested via leave-one-engine-out validation (mean RMSE on a completely unseen engine) to confirm Fleet Baseline Estimate degrades gracefully rather than failing outright.

Engine DNA / context-encoder ablation. Each engine's calibration parameters (its "Engine DNA") are fit in closed form from that engine's own train-split cycles only — verified leak-free by construction. We also built and tested a neural alternative: a PCA context encoder over each engine's earliest recorded sensor readings, feeding a downstream MLP diagnostic head.
